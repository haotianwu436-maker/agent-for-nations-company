# 已知限制（交付版）

## 1. 已完成能力

- 报告任务主链路可运行并可观测
- LLM 默认写作与 rule 回退可切换
- citation 绑定、覆盖率统计与告警收敛可用
- 固定七章节 markdown 输出可用
- 图表数据输出可用（bar/pie/line 取决于数据）

## 2. 仍为简化实现的部分

- 分类与标签识别仍以规则为主（非深度语义分类器）
- 抓取站点适配为通用策略，非站点级定制解析
- 检索增强（RAG）模块仅预留，未完成生产级接入

## 3. 依赖 LLM 的能力边界

- 仅章节摘要写作与建议生成依赖 LLM
- LLM 输出受模型稳定性和上下文质量影响
- 已有后处理清洗，但仍可能存在风格波动

## 4. 规则主导能力边界

- 抓取、去重、基础标签识别、citation 主绑定走规则链路
- 规则链路稳定但表达灵活性低于 LLM

## 5. citation 精度边界

- 当前支持文本片段匹配、关键词反查、证据兜底窗口
- 对“高度改写摘要”可保证来源可追溯，但摘录片段未必是最佳语义对齐

## 6. 抓取稳定性边界

- 对静态正文页效果较好
- 对强反爬、登录态、重脚本渲染页面稳定性有限
- Crawl4AI 不可用时会 fallback 到 requests/bs4

## 7. 最适合使用场景

- 周报/月报内测、业务评审、客户演示
- 中等规模白名单来源（可控站点）
- 有明确固定章节结构的报告生产

## 8. 不建议直接用于的场景

- 高并发生产环境（尚未接入完善异步队列与扩展治理）
- 强监管场景下的自动发布（建议人工审校后发布）
- 强时效突发场景的大规模全网抓取

## 9. 下一阶段优化建议

- 引入异步任务队列与任务隔离
- 引入更强的语义检索与 citation 对齐策略
- 站点级抓取适配器与发布时间抽取增强
- 增加更完善的监控告警与审计日志
